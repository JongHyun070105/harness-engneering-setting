# PitchConnect Communication Protocol (v1.0)

본 문서는 투수와 포수 간에 오가는 모든 통신 패킷의 구조를 정의합니다. 하네스 에이전트는 이 규약을 엄격히 준수하여 코드를 작성해야 합니다.

## 1. 패킷 구조 (Binary Serialization)

데이터 전송 효율과 지연 시간 최소화를 위해 모든 패킷은 바이트 배열(Uint8List)로 전송됩니다.

| 오프셋 | 필드명 | 타입 | 설명 |
|---|---|---|---|
| 0 | Packet Type | Uint8 | 0: 사인(PitchCall), 1: 제어(Control), 2: 하트비트(Heartbeat) |
| 1 | Pitch Type | Uint8 | 0: 직구, 1: 커브, 2: 슬라이더, 3: 체인지업, 4: 스플리터, 5: 기타 |
| 2 | Location | Uint8 | 0: 가운데, 1: 몸쪽 높게, 2: 몸쪽 낮게, 3: 바깥쪽 높게, 4: 바깥쪽 낮게 |
| 3 | Intensity | Uint8 | 0: 보통, 1: 강하게, 2: 유인구 |

## 2. 응답 메시지 예시
- `[0, 1, 4, 1]`: (사인) 커브를 바깥쪽 낮게 강하게 던져라.
- `[2, 0, 0, 0]`: (하트비트) 연결 상태 확인용 패킷.
