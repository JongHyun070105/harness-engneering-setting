# 디자인 시스템 토큰 계약 (Design Token Contract)

이 규칙은 프로젝트의 디자인 일관성을 해치는 임의의 수치 입력을 원천 차단한다.

## 🎨 일관성 원칙

### 1. 시맨틱 토큰 강제
- `Colors.red`, `Colors.blue`와 같은 직접적인 색상 지정을 금지한다.
- 반드시 `context.colorScheme.primary` 등 테마에서 정의된 시맨틱 토큰을 사용한다.
- 프로젝트 전용 스타일 클래스(예: `AppTheme.colors`)가 있다면 그것만을 사용한다.

### 2. 매직 넘버(Magic Numbers) 금지
- 위젯 사이의 간격으로 `13.5`, `27` 등 임의의 숫자를 사용하지 않는다.
- 반드시 4의 배수 또는 프로젝트 지정 스페이싱 토큰(예: `AppSpacing.md`)을 사용한다.

## 📏 레이아웃 제약
- 하드코딩된 `width`, `height`는 아주 특수한 경우(이미지 에셋 등)를 제외하고 지양한다.
- 대신 `LayoutBuilder`, `Flexible`, `Expanded`를 활용하여 반응형 구조를 만든다.

---

## 위반 시 조치
- 하드코딩된 값 발견 시 에이전트는 즉시 작업을 중단하고 사용자에게 **"디자인 시스템 내의 어떤 토큰을 사용해야 합니까?"**를 질문한다.
- 적절한 토큰이 없을 경우, 새로운 토큰 정의를 먼저 수행하고 구현에 착수한다.
