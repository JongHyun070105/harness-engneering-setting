# Flutter/Antigravity 프로젝트 체크리스트

> 각 항목에 ✅(충족) / ❌(미충족) / ➖(해당 없음)을 표시한다.
> 근거가 있으면 파일 경로를 함께 기록한다.

---

## 1. Agent Entry Point (P1) — 10항목

- [ ] `.agent/rules/global.md`가 존재하는가?
- [ ] 프로젝트 한 줄 요약이 포함되어 있는가?
- [ ] 기술 스택(Flutter 버전, 상태관리, 주요 패키지)이 명시되어 있는가?
- [ ] 빌드/테스트/린트 명령어가 복사-붙여넣기 가능한 형태인가?
- [ ] 디렉토리 구조 설명이 포함되어 있는가?
- [ ] 핵심 규칙(금지 사항, 필수 패턴)이 명시되어 있는가?
- [ ] **Step 2 '의도 명세(Intent Specification)' 템플릿을 준수하는가?**
- [ ] gotchas(실패 기록)가 참조되는가?
- [ ] 하위 프로젝트에 `.agent/rules/project.md`가 존재하는가?
- [ ] 환경 변수 설정 안내가 있는가?
- [ ] 최근 코드 변경을 반영하고 있는가?

---

## 2. 문서 구조 (P2, P5, P9) — 10항목

- [ ] `docs/` 디렉토리가 존재하는가?
- [ ] 코딩 컨벤션 문서가 존재하는가?
- [ ] 아키텍처 결정 기록(ADR)이 존재하는가?
- [ ] Progressive Disclosure 구조 (요약 → 상세 → 링크)인가?
- [ ] 문서 간 상호 참조(cross-reference)가 있는가?
- [ ] README.md에 프로젝트 개요가 있는가?
- [ ] 스킬이 SKILL.md → references/ 구조인가?
- [ ] 코드 내 주석이 "why"를 설명하는가?
- [ ] API 문서가 존재하는가?
- [ ] 문서에 실행 가능한 코드 예제가 포함되어 있는가?

---

## 3. Invariant 강제 (P3, P4, P10) — 12항목

- [ ] `analysis_options.yaml`에 린트 규칙이 설정되어 있는가?
- [ ] `dart analyze`가 체크포인트에서 실행되는가?
- [ ] `dart format`이 자동 적용되는가?
- [ ] 아키텍처 검증 스크립트가 존재하는가?
- [ ] **통합 검증 스크립트(harness_health.sh)가 존재하는가?**
- [ ] 테스트 실행이 프로토콜에 포함되어 있는가?
- [ ] pre-commit 워크플로우가 설정되어 있는가?
- [ ] `pubspec.lock`이 버전 관리되는가?
- [ ] Flutter/Dart SDK 버전이 고정되어 있는가?
- [ ] Conventional Commits 규칙이 정의되어 있는가?
- [ ] Good/Bad 예시가 컨벤션에 포함되어 있는가?
- [ ] import 순서 규칙이 있는가?
- [ ] 보안 검사(시크릿 하드코딩 체크)가 자동 실행되는가?

---

## 4. 아키텍처 (P6, P11) — 8항목

- [ ] 레이어 분리가 되어 있는가? (presentation/domain/data)
- [ ] 의존성 방향이 일관적인가?
- [ ] 순환 의존성이 없는가?
- [ ] feature 간 직접 import가 없는가?
- [ ] 공통 코드가 `core/`로 중앙화되어 있는가?
- [ ] 에러 처리 패턴이 일관적인가?
- [ ] 외부 의존성이 추상화 레이어를 통해 접근되는가?
- [ ] 테스트 구조가 소스 구조와 대칭적인가?

---

## 5. 관찰성 & 검증 (P8) — 8항목

- [ ] 체크포인트가 프로토콜에 포함되어 있는가?
- [ ] SDD 명세 대조 검증이 프로토콜에 있는가?
- [ ] gotchas 기록 루프가 있는가?
- [ ] 피드백 루프 소요 시간이 3분 이내인가?
- [ ] 에이전트가 테스트를 직접 실행할 수 있는가?
- [ ] 빌드 결과를 에이전트가 직접 확인할 수 있는가?
- [ ] 에러 메시지가 충분히 설명적인가?
- [ ] 실패 시 자동으로 gotchas에 기록하는 프로토콜이 있는가?

---

## 6. 운영 & 유지보수 (P7) — 6항목

- [ ] dead code 탐지 수단이 있는가?
- [ ] 불필요한 의존성이 정리되어 있는가?
- [ ] 문서가 코드 변경을 반영하고 있는가? (Doc Drift 없음)
- [ ] Stale 설정 파일이 정리되어 있는가?
- [ ] 정기적 리팩토링 프로세스가 있는가?
- [ ] 의존성 업데이트 프로세스가 있는가?

---

## 7. 코드 가독성 (P12) — 6항목

- [ ] 네이밍 컨벤션이 일관적인가?
- [ ] 매직 넘버가 상수로 추출되어 있는가?
- [ ] 파일이 적절한 크기인가? (300줄 이하)
- [ ] **이름 없는 TODO(// TODO:)가 없는가? (책임 소재 명시)**
- [ ] 함수가 적절한 크기인가? (50줄 이하)
- [ ] 조건문 중첩이 3단계 이하인가?
- [ ] 같은 유형의 파일이 일관된 내부 구조를 따르는가?
