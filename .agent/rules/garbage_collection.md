# Garbage Collection (GC) 정책

에이전트의 효율을 갉아먹는 주요 원인은 사용되지 않는 낡은 정보(Stale Info)입니다. 에이전트는 작업 수행 중 다음 GC 규칙을 인지하고 실행해야 합니다.

## 1. 노후 문서 식별 기준
- 만들어진 지 오래되었고, 더 이상 사용되지 않는 스킬이나 워크플로우 문서.
- 개발 환경이나 버전업으로 인해 더 이상 유효하지 않은 코드 방식이 적힌 `gotchas` 항목.

## 2. 폐기(Archive) 절차
- 낡은 문서를 발견한 에이전트는 임의로 삭제하지 않고, `.archive/` 폴더를 생성하여 해당 문서를 이동시킵니다. (예: `.archive/skills/old_skill.md`)
- 이동 후 `gotchas.md`나 사용자의 리뷰 기록에 "ㅇㅇ 문서는 만료되어 archive로 이동됨"을 명시합니다.

## 3. GC 트리거 시점
- 월 1회 또는 "프로젝트 청소해 줘", "Harness Audit 재수행" 요청이 있을 때 에이전트가 주도적으로 점검합니다.
