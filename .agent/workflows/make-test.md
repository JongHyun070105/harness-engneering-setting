# /make-test 워크플로우 구조 및 절차

사용자가 `/make-test [대상_파일]`을 요청하면, 당신은 테스트 엔지니어가 되어 다음 절차를 수행합니다.

## 1. 타겟 파일 및 환경 분석
- 사용자가 지정한 원본 파일의 내용(`view_file` 활용)을 분석합니다.
- 해당 파일 내의 핵심 비즈니스 로직, 예외 발생 가능성(Edge Cases), 상태 변화 등을 파악합니다.
- Flutter/Dart 환경의 경우 대응되는 `test/` 디렉토리 하위의 경로를 스캐닝합니다. (예: `lib/features/auth/login.dart` ➡️ `test/features/auth/login_test.dart`)

## 2. 테스트 시나리오 기획
- 코드를 무작정 짜기 전에, 무엇을 테스트할지 시나리오를 나열합니다.
- 성공 케이스(Happy Path)와 실패 케이스(Unhappy Path), 한계값(Boundary conditions)을 모두 도출합니다.

## 3. 테스트 코드 작성
- 도출된 시나리오를 바탕으로 유닛 테스트(Unit Test) 혹은 위젯 테스트(Widget Test) 코드를 작성합니다.
- 100% 커버리지를 목표로 하되, 불필요한 테스트보다 핵심 로직 검증에 집중합니다.
- mock 데이터나 의존성 주입이 필요한 경우 `mockito` 혹은 `mocktail`의 활용법을 설계에 반영합니다.

## 4. 검증 (Validation)
// turbo
- `flutter test [생성된_테스트_경로]` 또는 `dart test [경로]` 명령어를 실행하여 작성한 테스트가 통과하는지 직접 확인합니다.
- 에러가 발생하면 로그를 보고 스스로 테스트 코드를 수정하는 실행 루프를 거칩니다.

## 5. 결과 보고
- 사용자에게 어떻게 테스트를 구성했는지 요약하고, 추가로 검증이 필요한 엣지 케이스가 있는지 질문합니다.
